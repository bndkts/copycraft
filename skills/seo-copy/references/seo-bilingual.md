## 1. Search intent first (the map)

A keyword is the words a person typed to express a need. Rank for the **need**, not the
string. Name the intent before you write — it decides the page type, the structure, and
the tone. Get this wrong and no amount of keyword placement saves the page.

| Intent | The searcher wants | Typical queries (EN / DE) | Lead with | Page type |
|---|---|---|---|---|
| **Informational** | An answer / to learn | "what is X", "how does X work", "X meaning" / „Was ist X“, „Wie funktioniert X“, „X erklärt“ | A direct answer in the first 1–2 sentences, then depth | Guide, glossary entry, blog post |
| **Commercial** | To compare before buying | "best X", "X vs Y", "X software", "X reviews" / „bestes X“, „X Vergleich“, „X Software“, „X Erfahrungen“ | The differentiator + proof; what sets you apart | Category / comparison / product page |
| **Transactional** | To act now | "buy X", "X pricing", "book X demo", "X free trial" / „X kaufen“, „X Preise“, „X Demo“, „X kostenlos testen“ | The action + a risk-reducer, high on the page | Pricing, signup, demo, checkout |
| *(Navigational)* | A specific brand/page | "Acme login", "Acme pricing" / „Acme Login“ | Get them where they're going fast | Brand page / login |

**Why intent beats keyword density:** a definition essay served to a transactional query
(„X kaufen“) makes a ready buyer hunt for the button; a hard pitch served to an
informational query („Was ist X“) makes a learner bounce, and the bounce tells the engine
the page missed. Matching intent is the highest-leverage SEO-copy decision.

**Mixed intent:** one query can host two intents (e.g. „Markenschutz“ = both "what is it"
and "who provides it"). Handle the dominant intent in the H1 and opening, then serve the
secondary intent lower (a "what does it cost / how to start" H2). Don't split the H1 in two.

## 2. English on-page keyword placement

Place the keyword where it helps a scanner and a snippet; cover the topic with variants
everywhere else. Natural reading always wins over an exact-match count.

| Element | Rule | Why |
|---|---|---|
| **H1** (one per page) | Primary keyword, read naturally, one promise | Strongest on-page signal and the first thing a human reads |
| **First 100 words** | Primary keyword once; the direct answer for informational intent | Front-loading serves the scanner and the SERP snippet |
| **H2 / H3** | Semantic variants + real reader questions — not the repeated keyword | Covers the topic; subheads double as a scannable outline |
| **Body** | Variants and related terms naturally; answer the sub-questions | Modern search rewards topical coverage over phrase repetition |
| **Image alt text** | Describe the image truthfully; include a keyword only if it fits | Accessibility first; alt text is not a keyword dump |
| **URL / slug** | Short, keyworded, hyphenated, lowercase | Readable, stable, shareable |
| **Internal links** | Descriptive anchor text (not "click here") | Helps both ranking and the reader's next step |

**Semantic variants, not stuffing.** For *brand protection software*, the natural family
is: brand protection platform, brand monitoring, takedown service, anti-phishing,
impersonation detection. Spread these across H2s and body. Repeating the exact phrase to
hit a "density" target is an outdated tactic that reads like a robot wrote it — and reads
badly is the better reason to stop.

**Keyword in the H1, read naturally — constructed examples:**
- ✓ "Brand Protection Software That Takes Fakes Down — Not Just Flags Them"
- ✓ "Brand Monitoring Built to Catch Impersonation Early"
- ✗ "Brand Protection Software — Best Brand Protection Software for Brand Protection"
  (stuffed; reads like a machine; helps nothing)

## 3. Meta title & description limits (both languages)

Meta tags are ad copy for the SERP. The title nudges rank a little and the click a lot; the
description rarely moves rank but heavily moves the click-through. Write them last, once you
know the page's promise.

| Tag | Limit | Must contain | Notes |
|---|---|---|---|
| **Meta title** | **≤ ~60 characters** | Primary keyword near the front + one hook/benefit | Google truncates by pixel width near ~60 chars; a title cut mid-word reads broken. Brand name optional at the end after a `\|`. |
| **Meta description** | **≤ ~155 characters** | A benefit + a CTA | Not a ranking factor, but the difference between a click and a scroll-past. Write like an ad, not a summary. |

Count characters, not words. German runs longer than English (compounds), so German meta
tags hit the ceiling faster — tighten compounds or split a phrase rather than letting the
SERP truncate. Each example below states its character count.

**English meta — constructed:**
- Title (53): `Brand Protection Software | Take Down Fake Sites Fast`
- Description (138): `Find the fake shops and phishing pages abusing your brand — and get them taken down, confirmed not just flagged. 1,200+ brands. Scan free.`

**German meta — constructed (typography correct):**
- Title (52): `Markenschutz im Netz | Fake-Shops & Phishing stoppen`
- Description (132): `Was ist Markenschutz im Internet? So erkennen Sie gefälschte Shops und Phishing-Seiten – und nehmen sie vom Netz. Jetzt informieren.`

Note the German description uses `„…“` if it quotes, a Halbgeviertstrich `–` **with
spaces** (never the English `—`), and a native CTA. See §4d.

## 4. German specifics

German SEO is not English SEO with translated words. These are the points tools get wrong.

### 4a. Compound vs. split keywords

German builds closed compounds where English uses two words, and **users search both
forms**. They type the closed compound *and* the split paraphrase:

| Closed compound (often the H1 term) | Split / paraphrase (fits body sentences) |
|---|---|
| Markenschutz | Schutz der Marke · die Marke schützen |
| Phishing-Schutz | Schutz vor Phishing · vor Phishing schützen |
| Markenüberwachung | die Marke überwachen · Überwachung der Marke |
| Website-Fälschung | gefälschte Website · Fake-Website |

How to cover both without forcing repetition:
- Put the **closed compound** in the H1 — it's usually the primary search term and the
  cleanest noun.
- Let the **split form** appear naturally in body sentences („…damit Sie Ihre Marke
  schützen…“) and in an H2.
- Hyphenate mixed compounds correctly: „Phishing-Schutz“, „Markenschutz-Software“ — never
  the Deppenleerzeichen „Markenschutz Software“ (✗, see §4d).
- Don't glue an English adjective string onto a German noun to chase a keyword
  („SEO-optimierte Markenschutz Software Lösung“ ✗). Write the German sentence (§4e).

### 4b. Umlaut and ae/oe/ue variants

Some users type `ä/ö/ü`; others type `ae/oe/ue` (and slugs/URLs typically use the ASCII
form). Search engines largely fold these together, so this matters less than it once did —
but coverage still helps where users genuinely type the ASCII form.

- **Visible copy:** always correct umlauts. „Markenüberwachung“, „gefälschte Shops“,
  „für“. Never misspell the on-page word to chase a spelling variant — that's a hard error.
- **Slug / URL:** use the `ae/oe/ue` transliteration: `markenueberwachung`,
  `gefaelschte-shops`. This is where the ASCII variant naturally lives.
- **Don't** create „Markenueberwachung“ as visible body text. It's a misspelling, not a
  keyword variant.

### 4c. Native CTAs in meta and on page

A German CTA is written native, never a word-for-word rendering of an English button. This
applies inside meta descriptions too — the CTA there must read native.

| English origin | Native German |
|---|---|
| Get started / now | Jetzt starten · Jetzt loslegen · Los geht's |
| Learn more | Mehr erfahren · Mehr dazu |
| Start free trial / Try for free | Kostenlos testen · 30 Tage kostenlos testen |
| Book / Request a demo | Demo vereinbaren · Demo anfragen · Live-Demo ansehen |
| Contact sales | Vertrieb kontaktieren · Kontakt aufnehmen |
| Get a quote | Angebot anfordern |

Craft: action verb first (Hol dir, Starte, Teste, Sichere dir, Buche); attach the benefit
(„Jetzt starten & 20 % sparen“ beats a bare „Jetzt starten“); never a vague „Klicken Sie
hier“ or „Weiter“. Match the CTA pronoun to the page's du/Sie. (Full map and du/Sie verb
pairs: the `headlines-and-ctas` skill, if the plugin is present.)

### 4d. Typography in meta (hard rules)

Meta tags are copy, so the same DIN 5008 / Duden mechanics apply. These are genuine errors,
not preferences:

- **Quotation marks:** `„…“` (or `»…«`), never straight `"…"` or English `"…"`.
- **Dashes:** Halbgeviertstrich `–` **with spaces** for an aside; never the English
  Geviertstrich `—`. Bis-Strich `–` without spaces for ranges (10–18 Uhr).
- **No „in 2024“** before a bare year → „2024“ / „im Jahr 2024“.
- **No Deppenleerzeichen** in compounds: „Social Media Plattform“ ✗ →
  „Social-Media-Plattform“ ✓; „Markenschutz Software“ ✗ → „Markenschutz-Software“ ✓.
- **Currency** after the amount with a space: „1.999,00 €“ (not „€1,999.00“, not „1,99€“).
- **Percent** with a space: „20 %“, not „20%“.
- **Numbers:** Dezimalkomma, Tausenderpunkt → „1.250.000,99 €“. Date „29.06.2026“ or
  „29. Juni 2026“, never US format.

### 4e. Never sacrifice idiomatic German to force a keyword

This is the skill's hard line. When the natural German sentence and the exact-match keyword
phrase fight, **the natural sentence wins** and you place the keyword somewhere it fits (an
H2, a body sentence, the slug, alt text). Forcing an English-shaped phrase onto German:

- reads translated and lowers trust — fatal for a security/compliance brand;
- often introduces a Deppenleerzeichen or a wrong case, i.e. an actual error;
- does **not** rank better — search engines model German morphology and synonyms, so the
  natural compound and its variants already cover the intent.

Constructed contrast:
- ✗ „SEO-optimierte Markenschutz Software Lösung für Unternehmen“ — Deppenleerzeichen,
  Floskel („Lösung“), reads machine-made.
- ✓ „Software, die Ihre Marke vor gefälschten Shops und Phishing schützt“ — natural, the
  keyword family („Marke“, „schützt“, „Phishing“) is present, and it reads like a human
  wrote it.

The test: **read the sentence aloud.** If a native speaker would never say it that way,
the keyword lost the page. Rewrite it natural and relocate the keyword.

## 5. Front-loaded answers (the inverted pyramid for search)

Roughly 80 % of readers scan rather than read, and the SERP often pulls the opening lines
into the snippet. So put the conclusion first:

- **Informational:** answer the question in the first 1–2 sentences, *then* explain. „Was
  ist Markenschutz im Internet? Markenschutz im Internet bedeutet, …“ — the answer leads.
- **Commercial:** the differentiator and proof lead; the definition (if any) comes after.
- **Transactional:** the action and the risk-reducer sit high; don't bury the button under
  prose.
- One idea per paragraph; 3–4 sentences max per web paragraph; an H2 every ~200–300 words.

A front-loaded page wins the featured snippet, the scanner, and the impatient buyer at
once. A page that winds up to its point loses all three.

## 6. Pre-publish gate
- [ ] Intent named; page type and tone match it (informational / commercial / transactional).
- [ ] One H1; primary keyword in it and in the first 100 words, read naturally.
- [ ] Answer front-loaded for informational intent (first 1–2 sentences).
- [ ] H2s carry variants + real questions, not the repeated keyword.
- [ ] Meta title ≤ ~60 chars (keyword near front); description ≤ ~155 chars (benefit + CTA).
- [ ] German: compound **and** split variants covered; umlauts correct in copy, `ae/oe/ue`
      only in slug/where users type it; CTAs native; typography (`„…“`, „20 %“,
      „1.999,00 €“, „–“ with spaces, no „in 2024“, no Deppenleerzeichen) correct.
- [ ] No idiomatic German bent to force a keyword (read it aloud).
- [ ] Every claim has real proof or a bracketed placeholder — nothing fabricated.
- [ ] du/Sie consistent with the brand brief throughout.
