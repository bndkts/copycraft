# Conversion Checklist — English Generation (self-contained)

Use this when generating new English copy with the `copywriting-en` skill. It restates
the essentials so this folder works on its own — you need no other file to use it. Read
it before your first draft; skim it again before you deliver.

All examples below are **original and constructed** for illustration (domain: brand
protection / anti-phishing, B2B security). Write your own lines; never reuse these
verbatim, and never paste copy from a real site.

## 1. The eight essentials

1. **Benefits over features — but earn it with the feature.**
   A feature is *what it is*; a benefit is *what it does for the reader*. Don't drop the
   feature — name it, then land the outcome.
   - Feature alone: "Automated 24/7 monitoring."
   - Benefit, earned: "You hear about a fake shop the hour it launches — not weeks later
     from an angry customer."

2. **Mirror the voice of the customer.**
   Use the audience's own words from reviews, sales calls, and support tickets — not
   internal jargon. Copy in the reader's vocabulary feels written *for them*. It is the
   cheapest way to sound relevant.

3. **Clarity beats cleverness.**
   Readers scan; they don't read. Front-load the point (inverted pyramid): the most
   important idea first, in plain words. A headline that needs a second read has already
   lost most of the page.

4. **One idea per section.**
   Each section earns its place by making one point. If a block argues two things, split
   it into two.

5. **Specific beats vague.**
   Replace adjectives with numbers, mechanisms, and proof. "Fast" → "live in under 5
   minutes." "Trusted by many" → "used by 1,200 security teams." Vague praise is
   invisible; specifics are memorable and believable.

6. **Show proof, don't claim trust.**
   Trust is demonstrated, not asserted. Numbers, named customers, certifications,
   screenshots, concrete guarantees. The more a market fears being burned (security,
   finance, health), the more proof must carry the copy. **Never fabricate it** — use a
   bracketed placeholder when you don't have the real figure.

7. **Every CTA = action verb + benefit, matched to the reader.**
   "Get started" < "Start protecting your brand." Lead with a verb, attach the payoff,
   match the page's formality. One primary action per view.

8. **Reduce risk at the point of action.**
   Beside every ask, lower the cost of saying yes: "No credit card", "Cancel anytime",
   "Free for 30 days", "2-minute setup". The bigger the commitment, the more the
   friction-reducer matters.

## 2. Hero patterns

A hero = one-promise headline + mechanism/proof subhead + primary CTA + risk-reducer.

| Pattern | Shape | Example (constructed) |
|---|---|---|
| Outcome-first declarative | [Reader outcome], [sharpener] | "Fake shops taken down — not just flagged." |
| [Product] + verb | [Product] [verbs] [reader's thing] | "Acme removes phishing sites abusing your brand." |
| Imperative promise | [Verb] [reader's outcome] | "Protect your brand from impersonation." |
| Outcome + mechanism | [Outcome] — [how, in plain words] | "Catch impersonation early — automated detection, confirmed takedowns." |
| Problem → relief | [Reader's pain]. [Your fix]. | "Customers get scammed in your name. We find the fakes and shut them down." |

**Subhead job:** answer *how / for whom / prove it* in one scannable line. Extend the
headline; never repeat it.

## 3. Value-proposition patterns

- Lead with the **moment of value**, then the mechanism: "You find out the hour it
  happens (detection) → we file and confirm the takedown (mechanism)."
- Max ~3 value props per section; each a **distinct** outcome with no overlap.
- Each must *fail* the competitor copy-paste test — i.e., a competitor could **not**
  paste it onto their own site.

Weak → strong:
- ✗ "We provide best-in-class solutions for a wide range of brand security challenges."
- ✓ "You hear about a fake shop the moment it goes live — we detect it, report it, and
  confirm when it's gone."

## 4. Feature-to-benefit chain

For every feature, write three links and lead with the middle one:

| Link | Question | Example (constructed) |
|---|---|---|
| Feature | What is it? | "AI-assisted social monitoring" |
| Benefit | So what — for me? | "Impostor accounts get flagged within the hour" |
| Proof | Why believe it? | "[avg. detection time] across [N] brands" |

Deliver as **benefit (lead) → feature (name it) → proof (back it):**
"Impostor accounts get flagged within the hour — our monitoring watches every major
platform, with an average detection time of [X]."

## 5. Native English CTA do/don't pairs

Action verb + benefit, one primary action, risk-reducer alongside. All constructed.

| ✗ Weak / generic | ✓ Verb + benefit | Risk-reducer to pair |
|---|---|---|
| "Learn more" | "See how takedowns work" | — |
| "Submit" | "Scan my brand now" | "Results in minutes" |
| "Sign up" | "Start protecting your brand" | "No credit card" |
| "Get started" | "Start free monitoring" | "Free for 30 days" |
| "Contact us" | "Book a 20-minute demo" | "Pick a time that suits you" |
| "Request a quote" | "Get my custom quote" | "No obligation" |
| "Click here" | (never — name the action) | — |
| "Download" | "Get the brand-threat checklist" | "Free PDF, no email wall" |

Rules:
- One **primary** CTA per view; secondary links stay visually lighter.
- Match formality to the page — don't mix a casual and a formal ask.
- Never a translated-feeling "Make a request" — write the action a native speaker says.

---

## 6. The competitor copy-paste test

The fastest filler-detector: **could a competitor paste this exact sentence onto their
own site without it becoming false?** If yes, it says nothing specific to this brand.
Replace it with a number, a named capability, a mechanism, or a proof point. Run it on
every hero line, value prop, subhead, and trust statement — it works the same in English
and German.

---

## 7. Pre-delivery gate

- [ ] Each line passes the competitor copy-paste test.
- [ ] Every claim has real proof or a clearly bracketed placeholder — nothing invented.
- [ ] One idea per section; the point is in the first line.
- [ ] Benefits lead; features are named, not buzzword-dropped.
- [ ] Each CTA = verb + benefit, one primary action, risk-reducer nearby.
- [ ] Voice matches the brief's adjectives and anti-adjectives.
- [ ] No uncut AI-vocabulary (delve, leverage, robust, seamless, world-class…) unless one
      is genuinely the most accurate word.
